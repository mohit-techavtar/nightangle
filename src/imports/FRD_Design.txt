Part 1: UI Design System & Guidelines
This section defines the complete design system for the Tenant & License Management module. All screens, components, and interactions must adhere to these specifications to ensure visual consistency, accessibility, and a professional enterprise-grade experience.

1.1 Design Principles
Clarity First: Every screen must communicate its purpose within 3 seconds of viewing. Use clear hierarchy, labels, and visual grouping.
Data Density with Breathing Room: Enterprise users need to see a lot of data. Use compact tables with adequate padding, collapsible sections, and progressive disclosure.
Action Proximity: Primary actions must be within 1 click of the context they affect. No multi-page wizards for simple operations.
Status Visibility: System status (tenant state, subscription health, license balance) must be visible at all times through persistent indicators.
Consistent Feedback: Every user action must produce visible feedback within 200ms (loading states, success toasts, error highlights).
Accessibility: WCAG 2.1 AA compliance. Minimum contrast ratio 4.5:1 for text. All interactive elements must be keyboard-navigable.
